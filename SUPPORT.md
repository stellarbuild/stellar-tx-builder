# Support

This document describes how to get help with `stellar-tx-builder`.

---

## Before Asking for Help

Please try the following first:

1. **Read the documentation** — start with the [README](README.md) and the [`docs/`](docs/) directory.
2. **Check the [FAQ](FAQ.md)** — your question may already be answered.
3. **Search existing issues** — [https://github.com/stellarbuild/stellar-tx-builder/issues](https://github.com/stellarbuild/stellar-tx-builder/issues)
4. **Check the [Troubleshooting guide](docs/troubleshooting.md)** for common errors and resolutions.

---

## Support Channels

### 💬 GitHub Discussions (Recommended)

For questions, ideas, and general discussion:

👉 [github.com/stellarbuild/stellar-tx-builder/discussions](https://github.com/stellarbuild/stellar-tx-builder/discussions)

Use the following categories:
| Category | Purpose |
|---|---|
| **Q&A** | Ask questions about using the library |
| **Ideas** | Suggest features or improvements |
| **Show and Tell** | Share projects built with stellar-tx-builder |
| **General** | Anything that doesn't fit the above |

### 🐛 GitHub Issues

For confirmed bugs and feature requests only — not for support questions.

- [Report a bug](https://github.com/stellarbuild/stellar-tx-builder/issues/new?template=bug_report.yml)
- [Request a feature](https://github.com/stellarbuild/stellar-tx-builder/issues/new?template=feature_request.yml)
- [Report a documentation issue](https://github.com/stellarbuild/stellar-tx-builder/issues/new?template=documentation_issue.yml)

### 📧 Email

For private matters, security concerns, or partnership inquiries:

**hello@stellarbuild.io**

> For security vulnerabilities, please follow the [Security Policy](SECURITY.md) instead of emailing directly.

## Response Time Expectations

| Channel | Expected Response |
|---|---|
| GitHub Discussions (Q&A) | 1–5 business days |
| GitHub Issues | 3–7 business days for triage |
| Email | 3–5 business days |

We are a small team. Response times may be longer during holidays or high-activity periods. Community members are also encouraged to help answer questions in Discussions.

---

## Enterprise Support

If your organisation requires dedicated support, SLAs, or custom consulting
around building on the Stellar network, please contact us at
**hello@stellarbuild.io** to discuss options.
